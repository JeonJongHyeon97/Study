# 객체지향의 사실과 오해

### 협력(collaboration)

객체간 **요청**(request)와 **응답**(response)로 협력한다.

요청과 응답의 의사소통 수단 = **메시지**(message)

### 역할(role) & 책임(responsibility)

객체들은 협력하는 과정 속에서 특정한 역할과 그에 따른 책임을 갖고 있다.

- 여러 객체가 동일한 역할을 수행할 수 있다.
- 역할은 **대체 가능성**(substituable)을 의미한다.
- 책임을 수행하는 방법은 자율적으로 선택할 수 있다 → **다형성**(polymorphism)
- 한 객체가 동시에 여러 역할을 수행할 수 있다.

# 객체(object)

### 두가지 덕목

객체는 애플리케이션의 기능을 구현하기 위한 존재로 좋은 객체는 좋은 협럭을 이룬다.

객체는 다음의 두가지 덕목을 갖추어야 하며, 두 덕목 사이에서 균형을 유지해야 한다.

- **협력적** : 객체는 필요시 다른 객체에 요청을 해야하며, 다른 객체로 부터의 요청에 응답해야 한다.
- **자율적** : 객체는 공동의 목표 달성을 위해 협력에 참여(요청과 응답)하지만 스스로의 결정과 판단에 따라 행동해야 한다.
    - 객체의 자율성은 객체의 내부와 외부를 명확하게 구분(사적인 부분 간섭할 수 없도록 차단)하는 것으로부터 나온다. 객체는 다른 객체가 무엇을 수행하는지는 알 수 있지만 어떻게 수행하는지에 대해서는 알 수 없다.
    - 객체의 외부에서는 접근이 허락된 수단을 통해서만 객체와 의사소통해야 한다.
    - 자율성👍 → 유지보수, 재사용성 👍

### [**상태(state)**](#상태) & [행동(behavior)](#행동)

객체는 **협력**에 참여하기 위해 어떤 행동을 해야 한다면 그 행동을 하는 데 필요한 상태도 함께 지니고 있어야 한다 → 상태와 행위를 하나아의 단위로 묶는 자율적인 존재

객체의 상태를 변경하는 것은 객체의 자발적인 행동뿐이다.

### 메서드(method)

객체가 수신된 메시지를 처리하는 방법을 메서드(함수 또는 프로시저)라고 부른다.

메시지(외부요청)와 메서드(외부 요청 처리 방법)를 분리하는 것은 객체의 자율성을 높이는 핵심 메커니즘이다 → **캡슐화**(encapsulation)

# 객체지향이란?

- 객체지향이란 시스템을 상호작용하는 **자율적인 객체들의 공동체**로 바라보고 객체를 이용해 시스템을 분할하는 방법
- 자율적인 객체란 **상태**와 **행위**를 함께 지니며 스스로 자기 자신을 책임지는 객체를 의미
- 객체는 시스템의 행위를 구현하기 위해 다른 객채와 **협력**
- 각 객체는 협력 내에서 정해진 **역할**을 수행하며 역할은 관련된 **책임**의 집합
- 객체는 다른 객체와 협력하기 위해 **메시지**를 전송하고, 메시지를 수신한 객체는 메시지를 처리하는 데 적합한 **메서드**를 자율적으로 선택

### 객체와 클래스(class)

객체지향은 클래스를 지향하는 것과 다르다. 

객체지향의 중심에는 클래스가 아니라 객체가 위치하며, 중요한 것은 클래스들의 정적인 관계가 아니라 메시지를 주고받는 객체들의 동적인 관계이다.

# 상태

특정 시점에 객체가 가지고 있는 정보의 집합으로 객체의 구조적 특징을 표현

(객체지향에서 객체는 다른 객체의 상태에 직접 접근, 변경할 수 없다.)

= 정적 프로퍼티 + 동적 프로퍼티

= 단순한 값 + 링크

**프로퍼티**(property) : 객체의 상태를 구성하는 모든 특징. 프로퍼티는 고정되기 때문에 ‘정적’이지만, 프로퍼티 값(property value)은 변경되기 때문에 ‘동적’이다.

**속성**(attribute) : 객체의 구성하는(프로퍼티) 단순한 값

**링크**(link) : 객체와 객체 사이의 의미 있는 연결.객체의 식별자를 알고 있다는 뜻이며, 링크를 통해서만 메시지를 주고 받을 수 있다.

### 캡슐화(encapsulation)

객체는 상태를 외부에 노출하지 않으며, 외부에 노출되는 것은 행동뿐이다.

외부의 객체는 메시지를 수신한 객체가 상태를 어떻게(자율적 판단) 변경하는지, 실제 상태가 어떻게 변경되었는지 알 수 없다(간섭할 수 없다).

⇒ 객체의 자율성을 높이고 협력을 단순하고 유연하게 만든다.

# 행동

객체의 상태를 변경하거나 조회할 수 있는 방법으로 부수효과(side effect)를 초래한다.

- 객체의 행동은 상태에 영향을 받는다.
- 객체의 행동은 상태를 변경시킨다.
